# 风险、限制与合规清单

## 上游明确限制

上游 README 的“安全性与局限性”可以概括为：

1. 研究目的发布，未经额外安全措施不建议直接生产部署。
2. 可能产生不准确、有偏见或不适当输出。
3. 世界知识、真实品牌、人物、地标、复杂上下文理解仍弱于强闭源系统。
4. 图生图在严格保持主体、身份、布局和精细细节方面不稳定。
5. 长文本、密集排版、小字号、复杂设计布局仍可能错字、缺字或漂移。
6. 多人互动、遮挡、夸张姿势等复杂身体结构可能异常。
7. 小人脸、小肢体、眼睛和文字等细节可能有伪影。
8. 开源范围有限，训练和系统细节没有全部开源。

## 许可证与归属

上游仓库许可证为 Apache-2.0。一般含义：

- 可以使用、修改、分发。
- 需要保留许可证和版权/归属声明。
- 不提供担保。
- 不自动授予商标使用权。

注意：模型权重页、依赖库、第三方素材、用户上传图片和生成结果可能有各自的约束，不能只看 GitHub 仓库许可证就认为所有商业风险都消失。

## 商业服务提醒

上游 README_CN 特别声明：Boogu 团队目前没有推出任何针对 Boogu-Image 的收费 API、订阅或商业化服务。  

因此如果看到第三方以 Boogu-Image、booguimage、Boogu Image、Boogu 等名称销售服务，应按非官方服务处理，采购前需要核实：

- 服务方是否有清晰主体。
- 是否说明模型来源和许可证。
- 是否有隐私政策。
- 是否保存用户上传图片。
- 是否有内容审核和退款/售后机制。
- 是否误导性暗示官方授权。

## 产品化风险

| 风险 | 具体表现 | 缓解措施 |
| --- | --- | --- |
| 内容安全 | 色情、暴力、违法、歧视、政治敏感等输出 | prompt 审核、图像审核、人工复核、黑白名单 |
| IP/商标 | 生成真实品牌、卡通角色、名人肖像 | 禁用词、品牌授权检查、商用前人工审核 |
| 人像权 | 用户上传他人照片做编辑 | 上传声明、用途限制、审核、删除机制 |
| 文字错误 | 海报中文字错字、漏字、变形 | OCR 自动检查 + 人工复核 |
| 商品误导 | 商品图生成与实物不一致 | 标注 AI 生成、只做背景/场景辅助、人工确认 |
| 隐私泄露 | prompt 或图片发给 DashScope 等远程服务 | 默认本地处理；远程 rewrite 需显式开关和告知 |
| 稳定性 | 黑图、OOM、长时间无响应 | baseline preset、任务超时、失败重试、显存监控 |
| 成本失控 | 2K/Base/Edit 批量任务占满 GPU | 队列、限流、任务预算、按尺寸/模型定价 |

## 技术限制清单

上线前至少检查：

- [ ] 模型版本是否固定。
- [ ] 权重来源是否固定并记录 SHA。
- [ ] 每个 preset 是否有最大尺寸、最大 batch、最大步数限制。
- [ ] 是否禁止用户任意传入全部 CLI 参数。
- [ ] 是否记录 seed、prompt、rewrite prompt、模型版本和参数。
- [ ] 是否有超时和 OOM 恢复。
- [ ] 是否有输出内容审核。
- [ ] 是否有用户上传图删除策略。
- [ ] 是否有第三方 API key 管理，不把 key 写进命令日志。
- [ ] 是否有许可证和 attribution 页面。

## 评测建议

建议建立自己的小评测集，而不是只看 README 展示图：

| 场景 | 样本数 | 检查点 |
| --- | ---: | --- |
| 白底商品图 | 30 | 主体完整、阴影自然、背景干净 |
| 商品背景替换 | 30 | 主体一致、边缘融合、无多余物体 |
| 中文海报 | 50 | OCR 准确率、布局、字体一致 |
| 英文海报 | 30 | 拼写、大小写、排版 |
| 图像删除 | 30 | 删除干净、背景补全自然 |
| 改色/改材质 | 30 | 局部准确、不影响其他部分 |
| 人像/手部 | 30 | 身体结构、面部自然度 |
| 禁止内容 | 50 | 审核拦截率 |

只有当内部评测能稳定通过，才适合进入真实业务流。
